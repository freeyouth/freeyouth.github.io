Unity Android&IOS 命令行打包

by freeyouth — on Unity , Android , IOS 18 Aug 2015

Unity导出Android&IOS工程

一个导出Xcode工程的例子，Android工程类似。 CommandBuild.cs

static public void BuildXcodeProject()
{
	//设置编译参数
	PlayerSettings.SetScriptingDefineSymbolsForGroup(BuildTargetGroup.iPhone, "debug");
	//选择需要编译的场景
	string[] levels = { "Assets/Resources/Scene/Test1.unity", "Assets/Resources/Scene/Test2.unity" };
	//选择BuildTarget目标平台，xcode,android或者windows
	BuildPipeline.BuildPlayer(levels, Ios_Project_Dir, BuildTarget.iPhone, BuildOptions.None);
}
Unity -quit -batchmode -projectPath ${ProjectPath} -executeMethod CommmandBuild.BuildXcodeProject

Android工程导出apk

1、ant方式，使用android update生成build.xml。

//android list target 可列举出当前可用的target，-n 工程名，-p 工程路径
android update -n $name -p $path -t target

2、gradle方式,新的项目都使用这种方式了。

//工程gradle修改
sigingConfigs{
	releaseKeystore{
		keyAlias 'xxx'
		keyPassword '1111111'
		storeFile file('c:/xxx/release.keystore')
		storePassword '111111'
	}
}
buildTypes{
	release{
		...
		siginingConfig signingConfigs.releaseKeystore
	}
}
//命令行执行
gradle projectname:assembleRelease

IOS xcode工程导出ipa

1、编译出.app文件，从.app打包ipa

cd ${XCODE_PATH}
xcodebuild -target Unity-iPhone clean
xcodebuild -target Unity-iPhone -configuration Release
xcrun -sdk iphoneos PackageApplication -v $XCODE_PATH/build/${APPNAME} -o $WORK_PATH/${IPANAME}.ipa

2、从archive导出ipa。(貌似比从.app打包的ipa要小很多…)

//清理
cd ${XCODE__PATH}
xcodebuild clean -project ./Unity-iPhone.xcodeproj -configuration ${CONFIGURATION} -alltargets
//archive
xcodebuild archive -project ./Unity-iPhone.xcodeproj -scheme "Unity-iPhone" -archivePath sanguo.xcarchive -configuration Release
//从archive导出ipa
xcodebuild -exportArchive -archivePath sanguo.xcarchive -exportPath ${WORK_PATH}/${IPANAME}.ipa -exportFormat ipa -exportProvisioningProfile ${CODESIGN}
